# Getting help

Different kinds of question, different best places to land:

| Type | Where |
|---|---|
| "How do I set this up locally?" | [`docs/runbooks/local-dev.md`](docs/runbooks/local-dev.md) → if still stuck, [a Discussion](https://github.com/twarc-net/twarc/discussions/categories/q-a) |
| "I think I found a bug" | [Open an issue](https://github.com/twarc-net/twarc/issues/new?template=bug_report.yml) |
| "I have a feature idea" | Small + obvious → [issue](https://github.com/twarc-net/twarc/issues/new?template=feature_request.yml). Bigger / opinionated → [Discussion](https://github.com/twarc-net/twarc/discussions/categories/ideas) |
| "Question about the codebase / how a part works" | [Discussions → Q&A](https://github.com/twarc-net/twarc/discussions/categories/q-a) |
| "Want to chat about the roadmap" | [Discussions → General](https://github.com/twarc-net/twarc/discussions/categories/general) |
| "I found a security vulnerability" | [SECURITY.md](SECURITY.md) — please **do not** open a public issue |
| "I want to contribute code" | Start with [CONTRIBUTING.md](CONTRIBUTING.md), then pick a [`good first issue`](https://github.com/twarc-net/twarc/issues?q=is%3Aopen+is%3Aissue+label%3A%22good+first+issue%22) |
| "Content moderation on twarc.net" | Use the in-app report button on the live site, not GitHub |
| "Press / partnerships / something else" | `hello@twarc.net` |

## Response times

This is a small project run by a small team in their spare time, not a
funded company. Realistic expectations:

- Issues / Discussions: usually a response within ~3 days
- PRs: review within ~3 days, merge depends on scope
- Security reports: acknowledged within 72 hours (see [SECURITY.md](SECURITY.md))

If something has gone stale for over a week with no reply, please bump
the thread — it probably slipped through.
